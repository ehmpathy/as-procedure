## defect 1: TS2554 Error cause not supported

### what

type check failed with:

```
jest.integration.env.ts(105,7): error TS2554: Expected 0-1 arguments, but got 2.
```

### how (diff against origin/main)

declapract upgrade changed `jest.integration.env.ts` to use ES2022 Error cause:

```diff
-  } catch {
+  } catch (error) {
     throw new Error(
       `did you forget to \`npm run start:testdb\`? cant connect to database`,
+      { cause: error },
     );
   }
```

### why

the tsconfig.json had `"lib": ["DOM"]` which does not include ES2022 features. the Error constructor's second argument (options object with `cause`) was added in ES2022.

### fix

added ES2023 to lib array in tsconfig.json:

```json
"lib": ["DOM", "ES2023"]
```

ES2023 includes all ES2022 features including Error cause.

## defect 2: dpdm circular dependency in helpful-errors

### what

lint failed with:

```
• Circular Dependencies
  1) node_modules/.pnpm/helpful-errors@1.5.3/node_modules/helpful-errors/dist/HelpfulError.js -> node_modules/.pnpm/helpful-errors@1.5.3/node_modules/helpful-errors/dist/withHelpfulError.js
```

### how (diff against origin/main)

no direct diff — this is a transitive dependency issue. `simple-log-methods@0.6.9` depends on `helpful-errors@1.5.3` which has an internal circular dependency. dpdm detects cycles in node_modules.

### why

declapract upgrade added dpdm cycle detection to the lint command:

```json
"test:lint:cycles": "dpdm --no-warning --no-tree --exit-code circular:1 --exclude \"$(yq -r '.exclude | join(\"|\") // \"^$\"' .dpdmrc.yaml)\" 'src/**/*.ts'"
```

dpdm follows imports into node_modules and detects the cycle in helpful-errors@1.5.3.

### fix

upgraded simple-log-methods from 0.6.9 to 0.6.12:

```
rhx set.package.upgrade --package simple-log-methods --to 0.6.12
```

simple-log-methods@0.6.12 depends on helpful-errors@1.7.3 which does not have the cycle.

## defect 3: depcheck unused rhachet-roles-rhachet

### what

depcheck failed with:

```
Unused devDependencies
* rhachet-roles-rhachet
```

### how (diff against origin/main)

declapract upgrade added `rhachet-roles-rhachet@0.1.7` to devDependencies.

### why

this package provides rhachet roles but is not directly imported in code — it is used by the `rhachet init --roles` command during prepare. other similar packages (rhachet-roles-bhrain, rhachet-roles-bhuild, rhachet-roles-ehmpathy) were already in .depcheckrc.yml ignores but this new one was not.

### fix

added `rhachet-roles-rhachet` to .depcheckrc.yml ignores:

```yaml
  - rhachet-roles-ehmpathy
  - rhachet-roles-rhachet  # added
  - tsc-alias
```
