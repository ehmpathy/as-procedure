## defect 1: TS2554 Error cause

### classification: repo quirk

### evidence

the declapract practice at `src/practices/typescript/best-practice/tsconfig.json` does NOT set a `lib` array:

```json
{
  "extends": [
    "@tsconfig/strictest/tsconfig.json",
    "@tsconfig/node20/tsconfig.json"
  ],
  "compilerOptions": {
    // no lib array - inherits from @tsconfig/node20
  }
}
```

@tsconfig/node20 includes ES2022+ features by default.

this repo had a custom override:

```json
"lib": ["DOM"]
```

this override prevented ES2022 Error cause from availability.

### why repo quirk

- the practice does not set lib
- this repo chose to set lib: ["DOM"]
- the fix (add ES2023) is repo-specific
- other repos on the practice don't need this fix

### action

no upstream action needed. repo-specific fix applied.

## defect 2: dpdm circular dependency in helpful-errors

### classification: repo quirk

### evidence

simple-log-methods@0.6.9 depended on helpful-errors@1.5.3 which had a cycle.

### fix applied

replaced simple-log-methods with sdk-logs@0.9.2, which:
- uses helpful-errors@1.7.3 (no cycle)
- provides same LogMethods types
- adds cloudwatch outlet support

### why repo quirk

- this repo chose to adopt sdk-logs (breaks consumers: generateLogMethods → genLogMethods)
- other repos may not want this migration
- the practice still references simple-log-methods

### action

no upstream action needed. repo-specific migration to sdk-logs applied.

## defect 3: depcheck unused rhachet-roles-rhachet

### classification: practice bug

### evidence

```sh
gh search code --repo ehmpathy/declapract-typescript-ehmpathy 'rhachet-roles'
```

the practice at `src/practices/lint/best-practice/.depcheckrc.yml` has:

```yaml
ignores:
  - rhachet-roles-bhrain
  - rhachet-roles-bhuild
  - rhachet-roles-ehmpathy
  # absent: rhachet-roles-rhachet
```

the package.json upgrade adds `rhachet-roles-rhachet` to devDependencies, but the depcheckrc practice doesn't have it in ignores.

### why practice bug

- the practice adds the package
- the practice should add it to ignores
- all repos on the practice will hit this defect
- this is a defect in the practice, not a repo quirk

### action

file issue or pr to add `rhachet-roles-rhachet` to depcheckrc ignores in declapract-typescript-ehmpathy.
