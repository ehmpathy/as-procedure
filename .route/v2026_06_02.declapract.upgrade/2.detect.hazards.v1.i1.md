# 2.detect.hazards — iteration 1

## critical hazards

### 1. build:compile changed from tsdown to tsc (CRITICAL)

**what changed:**
- `build:compile` was `tsdown`
- now `build:compile` is `tsc -p ./tsconfig.build.json && tsc-alias -p ./tsconfig.build.json`

**why it may break:**
- this repo uses the **forwarded deps pattern** (see briefs: `pattern.forwarded-deps.md`)
- `tsdown.config.ts` exists and bundles types from `forwarded` array
- tsc does NOT bundle forwarded types — only tsdown does
- consumers will get TS2742 errors ("cannot be named without reference to...")

**how to verify:**
```sh
npm run build
# check dist/index.d.ts for `from 'simple-log-methods'` imports
# if present = BROKEN (types not bundled)
# if absent = OK (types bundled inline)
```

**verified:**
build fails with multiple TS errors:
- TS2835: relative imports need .js extensions (node16 moduleResolution)
- TS2307: cannot find module '@src/...' (path alias)

**root cause:**
tsconfig changed:
- `module`: commonjs -> node16
- `moduleResolution`: node -> node16

**required fix:**
restore `build:compile` to use tsdown (functional before upgrade)

## test hazards

### 2. slowtest reporter added to jest configs

**what changed:**
- jest configs now include `['test-fns/slowtest.reporter.jest', { slow: '10s', output: '.log/slowtest/unit.report.json' }]`

**why it may break:**
- requires test-fns 1.15.8+ which should include slowtest.reporter.jest
- test-fns was upgraded 1.4.2 -> 1.15.8

**how to verify:**
```sh
rhx git.repo.test --what unit --mode apply
```

**expected outcome:** pass if test-fns has the reporter
