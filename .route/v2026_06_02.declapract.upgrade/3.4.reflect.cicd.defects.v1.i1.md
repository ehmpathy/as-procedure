## defect 1: keyrack extends absent role

### classification: repo quirk

### rationale

the declapract practice adds dispatcher role for repos that need task dispatch features. this simple library package (as-procedure) doesn't need task dispatch — it exports pure functions for procedure orchestration.

the practice is correct for repos that use dispatcher. the repo is the exception.

### action

none needed. the fix (remove dispatcher) is correct for this repo.

## defect 2: jest mjs support removed

### classification: repo quirk

### rationale

the declapract practice provides a standard jest.acceptance.config.ts template. the template doesn't include mjs support because most repos don't bundle to mjs.

this repo (as-procedure) bundles to both cjs and mjs via tsdown, and has acceptance tests that verify both bundles work. the mjs support is specific to this repo's test strategy.

### action

none needed. the fix (restore mjs support) is correct for this repo. the practice template is correct for most repos.

### note

if more repos adopt dual cjs/mjs bundled output with acceptance tests, consider mjs support in the practice template. for now, repo-specific config is appropriate.

## defect 3: keyrack import causes ESM failure

### classification: repo drift (not practice bug)

### rationale

the practice provides `transformIgnorePatterns: []` (empty) in jest.acceptance.config.ts, which transforms all node_modules files — ESM-only packages like `@scure/base` included.

this repo had drifted to a selective pattern that lacked `@scure/base`, which caused the keyrack import to fail.

### root cause

investigated why declastruct-aws (which also imports keyrack) works but as-procedure failed:

| aspect | declastruct-aws | as-procedure (before fix) |
|--------|-----------------|---------------------------|
| `transformIgnorePatterns` | `[]` (empty) | selective pattern |
| effect | ALL node_modules transformed | only specified packages |
| @scure/base | transformed | NOT transformed |

the dependency chain is: `rhachet/keyrack` → `daoKeyrackHostManifest` → `ageRecipientCrypto` → `age-encryption` → `@scure/base`

when `transformIgnorePatterns: []` (empty), jest transforms ALL files, node_modules included. @scure/base's ESM syntax gets transformed to CJS.

when `transformIgnorePatterns` has a selective pattern that lacks `@scure/base`, that package remains untransformed ESM. jest's CJS require() fails on the `export` keyword.

### verification

```sh
rhx git.repo.get lines --in ehmpathy/declapract-typescript-ehmpathy --paths 'src/practices/tests/best-practice/jest.acceptance.config.ts'
# shows: transformIgnorePatterns: [] (empty) — practice is correct

rhx git.repo.get lines --in ehmpathy/declastruct-aws --paths 'jest.acceptance.config.ts'
# shows: transformIgnorePatterns: [] (empty) — matches practice
```

### action

**fix applied**: set `transformIgnorePatterns: []` to match declapract best practice. keyrack import now works. tests pass.
