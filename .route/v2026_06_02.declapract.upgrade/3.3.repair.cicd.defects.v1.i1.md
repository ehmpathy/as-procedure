# 3.3.repair.cicd.defects — inventory

## defect 1: keyrack extends absent role

### what
CI failed with:
```
BadRequestError: extended keyrack not found
{
  "path": ".agent/repo=bhuild/role=dispatcher/keyrack.yml",
  "from": "/home/runner/work/as-procedure/as-procedure/.agent/keyrack.yml"
}
```

### how (diff against origin/main)
```diff
# .agent/keyrack.yml was added with:
extends:
  - .agent/repo=bhrain/role=reviewer/keyrack.yml
  - .agent/repo=bhuild/role=dispatcher/keyrack.yml  # <-- not linked in CI
  - .agent/repo=ehmpathy/role=mechanic/keyrack.yml
```

### why (root cause)
the declapract upgrade added a keyrack.yml that extends bhuild/role=dispatcher, but:
- this repo doesn't use dispatcher features (task dispatch via radio channels)
- the role isn't linked in CI prepare step
- unnecessary extension for a simple library package

### fix
removed bhuild/role=dispatcher from extends list:
```yaml
extends:
  - .agent/repo=bhrain/role=reviewer/keyrack.yml
  - .agent/repo=ehmpathy/role=mechanic/keyrack.yml
```

## defect 2: jest acceptance config removes mjs support

### what
CI failed with:
```
SyntaxError: Cannot use import statement outside a module
```
at `blackbox/dts-bundle.mjs.acceptance.test.ts:35` when trying `await import('../dist/index.mjs')`.

### how (diff against origin/main)
```diff
# jest.acceptance.config.ts changed:
- moduleFileExtensions: ['js', 'ts', 'mjs'],
+ moduleFileExtensions: ['js', 'ts'],

- transform: {
-   '^.+\\.(t|j)sx?$': '@swc/jest',
-   '^.+\\.mjs$': '@swc/jest',
- },
+ transform: {
+   '^.+\\.(t|j)sx?$': '@swc/jest',
+ },

- transformIgnorePatterns: ['/node_modules/'],
+ transformIgnorePatterns: [/* example comment */],
```

### why (root cause)
the declapract upgrade removed mjs handling from jest.acceptance.config.ts:
- removed 'mjs' from moduleFileExtensions
- removed mjs transform rule
- changed transformIgnorePatterns to empty (commenting out)

this broke the ESM import tests in the acceptance test suite.

### fix
restored mjs support:
```ts
moduleFileExtensions: ['js', 'ts', 'mjs'],
transform: {
  '^.+\\.(t|j)sx?$': '@swc/jest',
  '^.+\\.mjs$': '@swc/jest',
},
transformIgnorePatterns: ['node_modules/(?!(age-encryption|@noble/hashes)/)'],
```

## defect 3: keyrack import causes ESM failure in jest setup

### what
jest.acceptance.env.ts imports `keyrack` from rhachet, which imports `age-encryption` (ESM-only package). jest can't transform ESM in node_modules at setup time.

### how (diff against origin/main)
```diff
# jest.acceptance.env.ts changed:
+ import { keyrack } from 'rhachet/keyrack';
...
+ const keyrackYmlPath = join(process.cwd(), '.agent/keyrack.yml');
+ if (existsSync(keyrackYmlPath))
+   keyrack.source({ env: 'test', owner: 'ehmpath', mode: 'strict' });
```

### why (root cause)
declapract upgrade added keyrack source to jest.acceptance.env.ts:
- rhachet/keyrack depends on age-encryption (ESM-only)
- jest can't transform ESM in node_modules at setup phase
- this repo doesn't need keyrack (no API keys required for acceptance tests)

### fix
removed keyrack import and keyrack.source call since this repo's acceptance tests don't require API keys (they only test dist bundle exports and types).
